# Security policy

## Supported versions

The project is pre-alpha. Only the latest code on the default branch is considered for security
fixes; no stable or long-term-support release exists.

## Reporting a vulnerability

Please use GitHub's private vulnerability-reporting or security-advisory feature for this
repository when it is available. If private reporting is unavailable, contact the repository owner
privately before opening a public issue. Do not include credentials, private datasets, account
identifiers, or exploitable details in a public report.

Include the affected revision, environment, reproduction steps, impact, and any suggested
mitigation. A response time is not guaranteed while the project remains pre-alpha.

## Operational scope

Version 0.1.0 has no exchange connectivity, credential handling, network market-data ingestion,
paper-trading service, or live order execution. If those capabilities are added later, the threat
model and this policy must be revised before they are enabled.
